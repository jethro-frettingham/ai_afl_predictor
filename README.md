# 🏉 AFL Predictor Bot — Discord Edition

An AWS-native bot that uses Claude AI to predict AFL match results each week
and posts predictions + results as rich embeds to your Discord server.

## How it works

| Day | What happens |
|-----|-------------|
| **Thursday 9am AEST** | Lambda fetches fixtures → Claude predicts each match → Discord embeds posted to #predictions |
| **Monday 9am AEST** | Lambda fetches scores → compares to predictions → Claude writes recap → Discord embeds posted to #results |

## Architecture

```
EventBridge (Thu) --> predict_matches Lambda --> Claude API (predictions)
                                             --> Discord webhook (embeds)
                                             --> DynamoDB (save predictions)

EventBridge (Mon) --> fetch_results Lambda  --> Squiggle AFL API (scores)
                                             --> DynamoDB (load predictions)
                                             --> Claude API (witty recap)
                                             --> Discord webhook (result embeds)
                                             --> DynamoDB (save results)

Both Lambdas --> Secrets Manager (API keys + webhook URLs)
```

## Discord setup (do this first)

1. Open your Discord server
2. Go to the channel you want predictions in > **Edit Channel** > **Integrations** > **Webhooks**
3. Click **New Webhook**, give it a name like "AFL Predictor" and optionally set an avatar
4. Click **Copy Webhook URL** — paste this into `terraform.tfvars`
5. Repeat for a second channel if you want predictions and results separated (optional)

That's it. No bot token, no OAuth, no developer portal needed.

## Prerequisites

- AWS account with CLI configured (`aws configure`)
- Terraform >= 1.5
- Python 3.12 + pip (for building the Lambda layer)
- Anthropic API key — https://console.anthropic.com

## Deployment

### 1. Build the Lambda layer

```bash
chmod +x build.sh
./build.sh
```

This packages `anthropic` and `requests` into a zip that gets deployed as a Lambda layer.

### 2. Configure

```bash
cd terraform
cp terraform.tfvars.example terraform.tfvars
# Fill in your Anthropic key and Discord webhook URLs
nano terraform.tfvars
```

### 3. Deploy

```bash
terraform init
terraform plan -var-file=terraform.tfvars
terraform apply -var-file=terraform.tfvars
```

Creates: 2 Lambda functions, 1 Lambda layer, 1 DynamoDB table, 1 Secrets Manager secret,
2 EventBridge rules, IAM role + policies, CloudWatch log groups.

### 4. Test immediately (don't wait for Thursday)

```bash
# Test predictions
aws lambda invoke \
  --function-name afl-predictor-predict-prod \
  --payload '{}' \
  --cli-binary-format raw-in-base64-out \
  out.json && cat out.json

# Test results
aws lambda invoke \
  --function-name afl-predictor-results-prod \
  --payload '{}' \
  --cli-binary-format raw-in-base64-out \
  out.json && cat out.json
```

## What gets posted to Discord

**Thursday — Predictions channel:**
- An opening embed announcing the round
- One embed per match with: teams, venue, date, predicted winner + margin, confidence bar (🟦🟦🟦⬜⬜), and Claude's reasoning
- A closing message

**Monday — Results channel:**
- A summary embed with Claude's witty recap and overall score (e.g. 7/9 correct)
- One embed per match showing the actual score, winner, and whether the prediction was ✅ correct or ❌ wrong
